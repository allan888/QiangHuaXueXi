# DFQL 论文 Q-Learning 方法总结

## 算法全称

**DFQL (Dynamic and Fast Q-learning)** — 动态快速Q-learning

## 核心思想

将经典Q-learning与人工势场法（APF）相结合，通过优化Q表初始化和奖励函数设计，解决无人水面艇（USV）在部分已知海洋环境中的离线和在线路径规划问题。

---

## 一、Q-learning 基础框架

### 1.1 标准Q-learning更新公式

```
Q(st, at) ← (1-α) * Q(st, at) + α * [rt+1 + γ * max_a Q(st+1, at+1)]
```

- **α**：学习率（0≼α≼1），本文取 **0.9**
- **γ**：折扣因子（0≼γ≼1），本文取 **0.9**
- **动作选择策略**：ε-greedy策略，基于Q表选择最优动作

### 1.2 状态空间与动作空间

- **状态**：二维笛卡尔网格坐标 (x, y)，环境为30×20网格
- **动作**：8个方向（Moore邻域）— 上、右上、右、右下、下、左下、左、左上
- **安全距离**：rsd = 5m，每格边长 = 2×rsd = 10m
  - 水平/垂直移动距离：10m
  - 对角移动距离：10√2 m

## 二、DFQL 三大核心改进

### 改进1：APF 初始化 Q 表

用人工势场法的势能值替代随机初始化的Q值，为USV提供**环境先验知识**，避免初期随机游走。

#### 离线模式（环境完全已知）

同时使用引力场（目标）和斥力场（障碍物）初始化Q表：

```
引力场：Ua1(st) = 1/2 * ka * ρg²(st)                    -- ka为引力比例因子
斥力场：Ur1(st) = 1/2 * kr * [1/ρob(st) - 1/ρo]²       -- kr为斥力比例因子，ρo为障碍物影响距离
总势能：Un1(st) = Ur1(st) + Ua1(st)
归一化：U1(st) = |Umax1 - Un1(st)| / Umax1
Q表初始化：Q01(s0, a0) = U1(st)
```

- 距离目标越近，Q值越大（引导USV向目标移动）
- 距离障碍物越近，Q值越小（引导USV远离障碍物）

#### 在线模式（环境部分已知/未知）

仅使用引力场（只知道目标位置，不知道障碍物）：

```
引力场：Ua2(st) = 1/2 * ka * ρg²(st)
总势能：Un2(st) = Ua2(st)                               -- 无斥力项
归一化：U2(st) = |Umax2 - Un2(st)| / Umax2
Q表初始化：Q02(s0, a0) = U2(st)
```

### 改进2：静态 + 动态混合奖励函数

将传统的常数奖励替换为静态奖励与动态奖励的组合。

#### 奖励公式

```
reward = rs * (1 + rd)
```

#### 静态奖励 rs（基于移动方向）

| 动作类型 | 奖励值 |
|----------|--------|
| 水平/垂直移动 | 1 |
| 对角移动 | √2 |
| 到达目标 | 10 |
| 碰到障碍物/边界 | -inf |

#### 动态奖励 rd（基于曼哈顿距离变化）

```
dt   = |ytarget - yt| + |xtarget - xt|         -- 当前状态的曼哈顿距离
dt+1 = |ytarget - yt+1| + |xtarget - xt+1|     -- 下一状态的曼哈顿距离
rd   = (dt - dt+1) / |dt - dt+1|                -- 归一化的距离变化
```

- 向目标靠近时 dt < dt+1, rd > 0 → **正奖励加成**
- 远离目标时 dt > dt+1, rd < 0 → **负奖励惩罚**
- 设计原理：奖励值随距离变化动态调整，引导USV持续向目标方向移动，避免无效随机探索

### 改进3：死胡同逃脱机制

改进算法使USV能从被障碍物阻挡的死胡同路径（dead-end path）中逃脱，避免陷入局部最小值无法到达目标。

---

## 三、算法工作流程

```
1. 离线模式下，基于APF初始化Q表（引力+斥力）
2. USV按离线规划路径行驶
3. 传感器检测到未知障碍物 → 切换到在线模式
4. 在线模式下，重新初始化Q表（仅引力），进行实时避障
5. 避开障碍物后 → 切换回离线模式
6. 重复步骤3-5，直到到达目标位置
```

---

## 四、与传统Q-learning的关键区别

| 方面 | 传统Q-learning (CQL) | DFQL |
|------|---------------------|------|
| Q表初始化 | 随机/零值 | APF势能值（含环境先验知识） |
| 奖励函数 | 静态常数 | 静态+动态组合 |
| 初始探索行为 | 随机游走 | 有方向地向目标移动 |
| 收敛速度 | 较慢 | 快速收敛 |
| 模式支持 | 单一 | 离线+在线切换 |

---

## 五、关键参数

| 参数 | 符号 | 值 | 说明 |
|------|------|-----|------|
| 学习率 | α | 0.9 | 控制新旧Q值的权重 |
| 折扣因子 | γ | 0.9 | 控制未来奖励的重要性 |
| 引力比例因子 | ka | 1.5 | APF中目标引力的缩放 |
| 斥力比例因子 | kr | 1.5 | APF中障碍物斥力的缩放 |
| 障碍物影响因子 | ρo | 2 | 障碍物斥力的影响范围 |
| 安全距离 | rsd | 5m | USV与障碍物的安全间距 |
| 最大迭代次数 | Max | 100 | 算法最大迭代次数 |

## 六、性能表现

在6种不同海洋环境（M01-M06）中与CQL、PSO、GWO、DA、MFO比较：

- **路径长度**：比CQL优化 **1.57%–3.13%**
- **转弯角度**：比CQL优化 **11.11%–55.56%**
- **计算时间**：比CQL优化 **2.91%–6.88%**
- **稳定性**：标准差在所有比较算法中最小
- **早熟性**：30次实验零早熟（对比PSO、GWO、DA、MFO均有早熟现象）

### 在线模式性能

与CQL、RRT、APF比较，在静态和动态障碍物环境下：
- 路径长度和转弯角度显著优于比较算法
- RRT随机性过大不适合USV路径规划
- APF在障碍物密集时无法计算出有效路径
- 局限性：计算时间较对比算法长，随地图大小呈指数增长

---

## 七、方法评价

### 优点

1. APF初始化Q表提供了有效的先验知识，显著减少初期随机探索
2. 静态+动态混合奖励引导USV持续向目标移动，加速收敛
3. 支持离线和在线两种模式切换，实用性强
4. 路径长度、转弯角度和稳定性均优于传统方法

### 缺点

1. 计算时间相比PSO、GWO等智能优化算法更长
2. 计算时间随地图规模呈指数增长
3. 仅支持单USV路径规划（未来工作方向为多USV）
