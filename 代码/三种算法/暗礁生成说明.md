# 暗礁（伏击障碍物）生成机制说明

## 三种障碍物概览

| 类型 | 颜色 | 位置 | 可探测性 |
|------|------|------|----------|
| 静态障碍物 | 蓝色 | 固定坐标，硬编码 | 始终可见 |
| 巡逻障碍物 | 橙色 | 固定航点间来回移动 | 始终可见 |
| 伏击障碍物（暗礁） | 紫色→品红 | 由路径决定 | 进入探测范围后才可见 |

## 暗礁生成细节

### 训练阶段

两种算法（Alg2 QL+ε-greedy / Alg4 SDP-QL）的训练环境均以 `n_ambush=0` 创建，**训练过程中不存在任何暗礁**。

```python
env2 = USVEnvironment(n_patrol=N_PATROL, n_ambush=0, seed=SEED)  # Alg2 训练
env4 = USVEnvironment(n_patrol=N_PATROL, n_ambush=0, seed=SEED)  # Alg4 训练
```

### 最终检测阶段（仅 Alg4）

只有 Alg4 的最终评估会引入暗礁，步骤如下：

1. **获取初步路径**：用训练好的 Q 表生成一条到达目标的路径 `prelim_path`
2. **在路径上放置暗礁**：调用 `place_ambush_on_path(prelim_path)`，位置规则：
   - 取路径约 **2/3 处**（第 404 行：`mid_idx = len(path_points) * 2 // 3`）
   - 在该点 ±3 个格子范围内**随机偏移**（第 408 行：`self.rng.uniform(-CELL_SIZE * 3, CELL_SIZE * 3)`）
   - 避开静态障碍物和巡逻障碍物的碰撞区域
3. **重新规划**：在有暗礁的环境下再次运行 SDP 路径规划，USV 进入探测范围（120m）后暗礁变为可见，需绕行

```python
prelim_path = alg4.get_path_sdp()          # 步骤 1
aob = env4.place_ambush_on_path(prelim_path) # 步骤 2
path4 = alg4.get_path_sdp()                # 步骤 3
```

### 位置是否固定

- **不固定**。暗礁位置依赖训练产出的路径，路径不同则暗礁位置不同
- 使用固定 seed=42 时，随机偏移是确定性的；但若训练参数或结果变化，暗礁位置随之变化
- Alg2 全程不使用暗礁，仅 Alg4 的最终评估使用

### 探测机制

- 探测距离：`DETECTION_RANGE = 120m`（第 59 行）
- USV 每步调用 `check_detection()`，当 USV 与暗礁中心距离 < 120m 时，暗礁 `detected = True`
- 未探测时以暗紫色半透明显示；探测后以品红色显示，障碍物可见，USV 开始规避
