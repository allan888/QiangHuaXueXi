# Fig.3~Fig.5 如何阅读

## 共同要素

- **横轴**：Episode 编号 $k$（$k = 1, 2, \dots, 2000$），表示第几次训练回合
- **浅色细线**：原始逐点数据 $x_k$
- **深色粗线**：滑动窗口平滑曲线，窗口宽度 $w = \max(1, 2000 \div 50) = 40$

$$\text{平滑值}(i) = \frac{1}{40}\sum_{j=i-39}^{i} x_j, \quad i \geq 39$$

- **蓝色 = Alg2（QL+ε-greedy）**，**绿色 = Alg4（SDP-QL）**

## Fig.3：Steps per Episode（步数收敛曲线）

### 纵轴计算公式

第 $k$ 个 episode 内的总步数 $S_k$：

$$S_k = \sum_{t=0}^{T_k} 1$$

其中 $T_k$ 为第 $k$ 个 episode 终止时的步数（到达目标或碰撞）。循环内每执行一次 `env.step(action)` 则 `steps += 1`。

| 终止原因 | $T_k$ 取值 |
|----------|-----------|
| 碰撞 | 碰撞发生时的步数（该步被计入） |
| 到达目标 | 到达目标时的步数 |
| 步数上限 | 最多 500 步（`GRID_SIZE × 5`） |

**代码对应**：

```python
for _ in range(max_steps):      # max_steps = 500
    action = choose_action(...)
    next_state, reward, done = env.step(action)
    steps += 1
    if done:
        break
self.episode_steps.append(steps)
```

### 如何读

| 现象 | 含义 |
|------|------|
| 曲线**下降**并稳定 | 算法在收敛，找到的路径越来越短、越来越高效 |
| 曲线**下降更快** | 收敛速度更快 |
| 曲线**最终稳定值更低** | 最终路径更短（步数少 = 路径高效） |
| 浅色散点**分布宽** | 该时间段内步数波动大，策略不稳定 |

## Fig.4：Cumulative Reward（累计回报收敛曲线）

### 纵轴计算公式

第 $k$ 个 episode 内所有步的即时奖励之和：

$$R_k = \sum_{t=1}^{T_k} r_t$$

#### Alg2 的 $r_t$ 构成

$$r_t = r_t^{\text{env}} + (d_{t-1}^{\text{usv}} - d_t^{\text{usv}}) \cdot \frac{\lambda_d}{W} - \lambda_{\text{step}}$$

其中：

| 符号 | 值 | 含义 |
|------|-----|------|
| $r_t^{\text{env}}$ | $\in \{-1, 0, +1\}$ | 碰撞 $-1$，正常移动 $0$，到达目标 $+1$ |
| $d_t^{\text{usv}}$ | 连续值 | 第 $t$ 步后 USV 中心到目标的欧氏距离 $\sqrt{(x_t - g_x)^2 + (y_t - g_y)^2}$ |
| $\lambda_d$ | $0.05$ | 距离奖励系数 |
| $W$ | $1000$ | 世界尺寸（用于归一化） |
| $\lambda_{\text{step}}$ | $0.001$ | 每步惩罚 |

**代码对应**：

```python
next_state, reward, done = env.step(action)  # reward ∈ {-1, 0, 1}

if reward >= 0:  # 未碰撞
    cur_dist = _distance_to_goal(usv.x, usv.y)
    reward += (prev_dist - cur_dist) * 0.05 / 1000
    prev_dist = cur_dist

reward += -0.001         # step_penalty
total_reward += reward
```

**具体取值示例**（Alg2）：

| 场景 | $r_t^{\text{env}}$ | 距离项 | 步惩罚 | $r_t$ |
|------|-------------------|--------|--------|-------|
| 碰撞 | $-1$ | $0$（跳过距离计算） | $-0.001$ | $-1.001$ |
| 正常移动，靠近目标 | $0$ | $+\frac{10 \times 0.05}{1000} \approx +0.0005$ | $-0.001$ | $\approx -0.0005$ |
| 到达目标 | $+1$ | 有距离项 | $-0.001$ | $\approx +0.999$ |

#### Alg4 的 $r_t$ 构成

$$r_t = r_t^{\text{env}} - \lambda_{\text{step}}$$

Alg4 不计算距离奖励，仅环境奖励 + 步惩罚。且碰撞的动作不会被计入 `total_reward`（碰撞后 continue，不执行 reward 累加）。

**代码对应**：

```python
for a, d in candidates:
    next_state, reward, done = env.step(a)
    if reward < 0:      # 碰撞 → 不计入累计回报, 尝试下一个候选
        continue
    else:
        reward += -0.001  # step_penalty
        total_reward += reward
        break
```

### 如何读

| 现象 | 含义 |
|------|------|
| 曲线**从负值上升** | 从频繁碰撞（$\approx -1$）逐步学到避开障碍物 |
| 曲线**稳定在正值** | 能持续到达目标（$+1$ 奖励抵消了步惩罚） |
| 曲线**收敛到 $\approx +1.0$** | 几乎无碰撞，稳定到达目标 |
| Alg2 上升**比 Alg4 慢** | ε-greedy 需要更多探索才学会，SDP 天然避障 |

### 注意

Alg2 和 Alg4 的 $r_t$ **计算公式不同**（Alg2 多了距离奖励项），因此两条曲线的纵轴不可直接比较绝对值。正确的读法是：**各自是否收敛、收敛趋势是否稳定**。

## Fig.5：Replanning Count（重规划次数收敛曲线）

### 纵轴计算公式

#### Alg2 的重规划次数

第 $k$ 个 episode 内发生碰撞的次数。由于一次碰撞即终止 episode，每个 episode 的取值只可能是 0 或 1：

$$N_k^{\text{Alg2}} = \sum_{t=1}^{T_k} \mathbb{1}[r_t^{\text{env}} < 0] \in \{0, 1\}$$

**代码对应**：

```python
for _ in range(max_steps):
    action = choose_action_egreedy(state)
    next_state, reward, done = env.step(action)
    if reward < 0:          # 碰撞
        replan_count += 1   # 计数
    # update Q, etc.
    if done:
        break
```

#### Alg4 的重规划次数

第 $k$ 个 episode 内，SDP **第一候选动作被拒绝**的次数：

$$N_k^{\text{Alg4}} = \sum_{\text{step}} \mathbb{1}[\text{SDP 候选 } (a_1, d_1) \text{ 执行后 } r^{\text{env}} < 0]$$

每步 SDP 按曼哈顿距离排序出 $(a_1, d_1), (a_2, d_2), (a_3, d_3)$ 三个候选，先尝试距离最近的 $a_1$：
- 若 $a_1$ 成功 → 不计数
- 若 $a_1$ 碰撞 → 计数 +1，换 $a_2$……直到找到第一个成功的动作

**代码对应**：

```python
candidates = rank_actions_sdp(state)  # [(a1, d1), (a2, d2), (a3, d3)]
first_try = True
for a, d in candidates:
    next_state, reward, done = env.step(a)
    if reward < 0:          # 该候选碰撞
        collision_count += 1
        if first_try:        # 是第一候选
            replan_count += 1
            first_try = False
        continue
    else:                   # 该候选成功
        # 更新 Q, 移动 USV, break
        break
```

### 如何读

| 现象 | 含义 |
|------|------|
| 曲线**从高值下降** | 算法在减少"选错动作" |
| 曲线**稳定在 0 附近** | 策略已稳定，几乎不再碰撞 |
| Alg4 **一开始就很低** | SDP 贪心天生避障，极少选错 |
| Alg2 **从 1 下降到趋近 0** | 前期纯随机探索必撞，后期 ε 衰减 + Q 值学习后几乎不撞 |

## 三图联合解读示例

假设跑出来的趋势如下：

| 图 | Alg2 | Alg4 |
|----|------|------|
| Fig.3 Steps | 从 $\sim 80$ 下降到 $\sim 40$，震荡较大 | 从 $\sim 36$ 稳到 $\sim 35$，几乎不波动 |
| Fig.4 Reward | 从 $\sim -1.0$ 上升到 $\sim +1.0$，爬坡缓慢 | 全程稳定在 $\sim +1.0$ |
| Fig.5 Replans | 从 $1$ 下降到 $\approx 0.05$ | 全程接近 $0$ |

**解读**：

1. **Alg4 收敛极快且稳定**：SDP 从第一个 episode 就能无碰撞到达目标，路径稳定
2. **Alg2 逐步学习但震荡大**：ε-greedy 前期碰撞率高（Fig.5 = 1），步数多（Fig.3 高），累计回报为负（Fig.4 负值）；随 epsilon 衰减逐步收敛
3. **最终两者都能完成任务**，但 Alg4 训练效率远高于 Alg2——这就是论文主张 SDP-QL 优于原始 QL 的原因
