# 评价指标说明

## 指标一：平均累计回报（Average Cumulative Reward）

### 定义

每 episode 的累计回报为从起始到终止（碰撞或到达目标）所有步的即时奖励之和：

$$R_{\text{episode}}^{(k)} = \sum_{t=0}^{T_k} r_t^{(k)}$$

其中：
- $k$：episode 编号（1, 2, ..., 2000）
- $T_k$：第 $k$ 个 episode 的步数
- $r_t^{(k)}$：第 $k$ 个 episode 中第 $t$ 步的即时奖励

**后 $N$ 个 episode 的平均累计回报**：

$$\bar{R}_N = \frac{1}{N} \sum_{k=K-N+1}^{K} R_{\text{episode}}^{(k)}$$

其中 $K$ 为总 episode 数（2000），$N=500$。

### 即时奖励构成

- **Alg2 (QL+ε-greedy)**：
  $$r_t = r_{\text{env}} + (d_{t-1} - d_t) \cdot \frac{\lambda_d}{\text{WORLD\_SIZE}} + r_{\text{step}}$$

- **Alg4 (SDP-QL)**：
  $$r_t = r_{\text{env}} + r_{\text{step}}$$

其中：
- $r_{\text{env}} \in \{-1 \text{ (碰撞)}, 0 \text{ (正常)}, 1 \text{ (到达目标)}\}$
- $d_t$：第 $t$ 步时 USV 到目标的欧氏距离
- $\lambda_d = 0.05$：距离奖励系数
- $r_{\text{step}} = -0.001$：每步惩罚

### 代码位置

- `QLearningBase.episode_rewards` 列表记录每 episode 的累计回报
- `main()` 中计算：`avg_reward2 = np.mean(alg2.episode_rewards[-500:])`
- 输出格式：`AvgReward: -0.xxxx`

## 指标二：收敛曲线（Convergence Curve）

### 定义

以 episode 为横轴，绘制每个 episode 的性能指标变化趋势，并用滑动窗口平滑：

$$\text{MA}_w(i) = \frac{1}{w} \sum_{j=i-w+1}^{i} x_j, \quad i \geq w-1$$

其中 $w = \frac{K}{50}$（约 40），$x_j$ 为第 $j$ 个 episode 的指标值。

共绘制三条收敛曲线：

| 图 | 指标 | 含义 |
|----|------|------|
| Fig.3 | Steps per Episode | 越小越好，表示路径更高效 |
| Fig.4 | Cumulative Reward | 越高越好，表示累计回报收敛到稳定正值 |
| Fig.5 | Replanning Events per Episode | 越低越好，表示策略越稳定 |

### 代码位置

- `main()` 中生成 Fig.3 / Fig.4 / Fig.5
- 原始数据：`alg.episode_steps` / `alg.episode_rewards` / `alg.episode_replans`
- 平滑：`np.convolve(data, np.ones(w)/w, mode='valid')`

## 指标三：重规划次数（Replanning Count）

### 定义

**Alg2 (QL+ε-greedy)** 的重规划：

$$N_{\text{replan}}^{(k)} = \sum_{t=0}^{T_k} \mathbb{1}[r_{\text{env},t} < 0]$$

即统计该 episode 内所有**碰撞事件**的次数。一次碰撞就意味着当前选择的动作导致路径失败，需要重新规划路径（重新选择动作或重置环境）。

**Alg4 (SDP-QL)** 的重规划：

$$N_{\text{replan}}^{(k)} = \sum_{\text{step}} \mathbb{1}[\text{首个 SDP 排名动作执行失败}]$$

SDP 按曼哈顿距离排序动作（最近优先），尝试执行排名第一的动作：
- 如果**第一候选失败**（碰撞）→ 计数 +1，尝试第二候选
- 如果**最终采用的不是第一候选**→ 也计为重规划事件

这衡量了 SDP 贪心启发式被物理碰撞约束"纠正"的频次。

### 总重规划次数

$$N_{\text{replan}}^{\text{total}} = \sum_{k=1}^{K} N_{\text{replan}}^{(k)}$$

### 训练碰撞总次数

$$N_{\text{collision}}^{\text{total}} = \sum_{k=1}^{K} \sum_{t=0}^{T_k} \mathbb{1}[r_{\text{env},t} < 0]$$

统计所有 episode 中所有导致碰撞的动作尝试次数。

### 代码位置

- `Alg2_OriginalQL_Dynamic.train()`：`replan_count += 1` 和 `collision_count += 1` 在 `reward < 0` 时
- `Alg4_ProposedQL_Dynamic.train()`：`replan_count += 1` 在首个 SDP 候选失败时；`collision_count += 1` 在任何候选失败时
- `QLearningBase.episode_replans` / `QLearningBase.episode_collisions` 存储每 episode 的计数

## 评价指标汇总表（运行输出示例）

```
 ======================================================================
  Evaluation Metrics
 ======================================================================
  Algorithm                       Train(s)   Steps  Success  AvgRew  Replans  Collis
 ------------------------------------------------------------------------------
  Alg2_OriginalQL_Dynamic           106.5      39      Yes  1.030      123     456
  Alg4_ProposedQL_Dynamic            85.2      36      Yes  0.998        8      12
```

**解读**：
- **Alg4** 的 Replans 远少于 Alg2，说明 SDP 启发式的动作选择比纯 Q 探索更稳定
- **Alg4** 的 Collis 也更少，说明 SDP 本身就能避开大部分碰撞，但训练早期仍有碰撞
- **Alg2** 的 AvgRew 可能更高（因为距离奖励塑形），但收敛曲线波动更大
